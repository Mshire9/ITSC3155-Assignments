# Sprint Review 1

## Sprint Dates
April 15 – April 22

## Sprint Goal
The goal of Sprint 1 was to set up the FastAPI project, connect it to the MySQL database, and implement the basic restaurant ordering features.

## Completed Work
- Set up FastAPI project
- Connected MySQL database
- Created Menu table and CRUD endpoints
- Created Orders table and CRUD endpoints
- Tested endpoints using Swagger UI (/docs)

## User Stories Completed
- As a customer, I want to view the menu so I can decide what to order
- As a customer, I want to place an order so I can buy food
- As a staff member, I want to manage menu items so the menu stays updated
- As a customer, I want to track my order so I know its status

## Challenges
- Had some difficulty connecting FastAPI to MySQL
- Ensuring endpoints matched project requirements

## What We Learned
- How to connect FastAPI with MySQL
- How CRUD operations work in APIs
- How to test endpoints using Swagger UI

## Summary
Sprint 1 was successful because the API was set up and core features were implemented.
